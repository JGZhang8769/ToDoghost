# 設計規範 (Design Guidelines)

## 1. 視覺主題 (Visual Theme)
整體應用程式採用「奶茶色系 (Milktea Palette)」為主軸，強調簡單、乾淨、溫暖的視覺體驗。

### 1.1 色彩計畫 (Color Palette)
使用 Tailwind 擴展色彩定義 (`milktea`):
*   `50` (`#fcfaf8`): 應用程式背景色，極淺的奶茶白。
*   `100` (`#f5efe6`): 區塊背景、輸入框底色。
*   `200` (`#eadbc3`): 邊框與分隔線。
*   `300` (`#dfc7a0`): 任務卡片底色 (日視圖)。
*   `400` (`#d1ab76`): 次要文字、未選取狀態。
*   `500` (`#c59454`): 裝飾性元素 (如月視圖的紅點指示器改用符合主題的顏色)。
*   `600` (`#b87e47`): 主要按鈕、選中狀態 (Primary Action)。
*   `800` (`#805336`): 次要標題、描述文字。
*   `900` (`#68452e`): 主要標題、強調文字。
*   **警告與緊急色**: 保留紅色 (`bg-red-500`, `text-red-500`) 用於緊急標籤、取消排程滑動背景與垃圾桶。
*   **操作色**: 藍色 (`bg-blue-500`) 用於複製操作的滑動背景。

## 2. 響應式與 PWA 設計 (Responsive & PWA Design)
*   以 iOS Mobile (iPhone 15 Pro / iPad) 為優先考量。
*   禁用原生網頁的反彈效果與捲動條 (`no-scrollbar`)。
*   使用 Tailwind 的 Flexbox 與 Grid 系統確保在不同尺寸螢幕下的佈局合理性。
*   實作底部抽屜 (Bottom Drawer) 供未排程事項使用，節省畫面空間並方便拖拉。
*   隱藏 Safari 導覽列的 PWA 專用配置 (`manifest.webmanifest` 內的 `display: standalone`)。

## 3. 互動設計 (Interaction Design)
*   **拖拉 (Drag and Drop)**:
    *   使用者可將未排程抽屜中的事項拖曳至月、週視圖的特定日期，或日視圖的特定時間區段。
    *   已排程的事項可透過拖曳至畫面右上角的垃圾桶來「取消排程」。
*   **滑動 (Swipe)**:
    *   在未排程列表與日曆視圖中的事項支援水平滑動。
    *   向左滑動顯示「複製」按鈕 (藍底)。
    *   向右滑動顯示「取消排程」按鈕 (紅底)。
*   **點擊展開**:
    *   月視圖中，點擊包含代辦事項的日期會展開浮動選單 (Popover) 顯示精簡列表，再次點擊精簡列表項目才會彈出詳細編輯視窗。

## 4. 向量圖示 (SVG Vectors)
為了增加趣味性與識別度，應用程式不使用純圖片，而是使用自行繪製的純 SVG 組件 (`app-svg-icon`) 來呈現使用者頭像：
1.  孟加拉豹貓 (Bengal Cat)
2.  金漸層貓 (Golden Chinchilla)
3.  兔子 (Rabbit)
4.  老虎 (Tiger)
